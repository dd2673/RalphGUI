# 快速开始：您的第一个 Ralph 项目

本教程引导您在现有项目上启用 Ralph 并运行您的第一个 autonomous 开发循环。到最后，您将拥有 Ralph 构建的简单 CLI todo 应用程序。

## 前置要求

- 全局安装 Ralph (`./install.sh` 从 ralph-claude-code repo)
- 安装 Claude Code CLI (`npm install -g @anthropic-ai/claude-code`)
- 一个项目目录 (我们将创建一个)

## 步骤 1：创建您的项目

让我们创建一个简单的 Node.js 项目：

```bash
mkdir todo-cli
cd todo-cli
npm init -y
git init
```

## 步骤 2：启用 Ralph

运行交互式向导：

```bash
ralph-enable
```

向导将：
1. 检测您的项目类型 (Node.js/TypeScript)
2. 询问任务来源 (您现在可以跳过)
3. 使用入门文件创建 `.ralph/` 目录

您将看到如下输出：

```
Ralph Enable Wizard
==================

Phase 1: Environment Detection
------------------------------
Detected project type: javascript
Detected package manager: npm
Git repository: yes

Phase 2: Task Source Selection
------------------------------
No task sources selected. You can add tasks manually.

Phase 3: Configuration
------------------------------
Creating .ralph/ directory structure...

Phase 4: File Generation
------------------------------
Created: .ralph/PROMPT.md
Created: .ralph/fix_plan.md
Created: .ralph/AGENT.md
Created: .ralphrc

Ralph is now enabled for this project.
```

## 步骤 3：自定义您的需求

`ralph-enable` 后，您有了需要自定义的入门文件。打开 `.ralph/PROMPT.md` 并替换占位符内容：

```markdown
# Ralph 开发指令

## 上下文
您是 Ralph，一个 autonomous AI 开发 agent，正在构建 Node.js 中的 CLI todo 应用程序。

## 当前目标
1. 创建具有 add、list、complete 和 delete 命令的命令行 todo 应用
2. 将 todos 存储在 JSON 文件中 (~/.todos.json)
3. 使用 commander.js 进行参数解析
4. 包含有用的 --help 输出
5. 使用 Jest 编写单元测试

## 关键原则
- 保持代码简单和可读
- 使用 async/await 进行文件操作
- 提供清晰的错误消息
- 遵循 Node.js 最佳实践
```

## 步骤 4：定义您的任务

编辑 `.ralph/fix_plan.md` 以列出特定任务：

```markdown
# 修复计划 - Todo CLI

## 优先级 1: 核心结构
- [ ] 使用依赖项设置 package.json (commander, jest)
- [ ] 使用 commander 设置创建 src/index.js 入口点
- [ ] 创建 src/storage.js 用于 JSON 文件操作

## 优先级 2: 命令
- [ ] 实现 `todo add "task description"` 命令
- [ ] 实现带状态指示器的 `todo list` 命令
- [ ] 实现 `todo complete <id>` 命令
- [ ] 实现 `todo delete <id>` 命令

## 优先级 3: 完善
- [ ] 为所有命令添加 --help 文档
- [ ] 处理边界情况 (空列表、无效 ID)
- [ ] 为存储模块编写单元测试
```

## 步骤 5：启动 Ralph

现在让 Ralph 构建您的项目：

```bash
ralph --monitor
```

这会打开一个 tmux 会话，包含：
- **左窗格**: Ralph 循环输出 (Claude 正在做什么)
- **右窗格**: 实时监控仪表板

### 您将看到的内容

Ralph 将：
1. 读取您的 PROMPT.md 和 fix_plan.md
2. 按优先级顺序开始实施任务
3. 创建文件、运行测试、更新 fix_plan.md
4. 继续直到所有任务完成

### 监控提示

- **Ctrl+B，然后 D** - 从 tmux 分离 (Ralph 继续运行)
- **tmux attach -t todo-cli** - 重新附加以观看进度
- **ralph --status** - 检查当前循环状态

## 步骤 6：查看结果

当 Ralph 完成 (或您想检查进度) 时，查看：

```bash
# 查看创建了哪些文件
ls -la src/

# 检查更新的 fix_plan.md
cat .ralph/fix_plan.md

# 运行 Ralph 编写的测试
npm test

# 试用您的新 CLI
node src/index.js add "Buy groceries"
node src/index.js list
```

## 刚才发生了什么？

Ralph 遵循此循环：
1. **读取** - 加载 PROMPT.md 获取上下文，fix_plan.md 获取任务
2. **实施** - 为最高优先级未选中任务编写代码
3. **测试** - 运行任何测试并修复失败
4. **更新** - 在 fix_plan.md 中标记已完成的任务
5. **重复** - 继续直到设置 EXIT_SIGNAL

## 常见问题

### Ralph 提前停止 - 为什么？

检查 `.ralph/logs/` 获取最新日志。常见原因：
- 达到速率限制 (等待重置)
- 断路器打开 (检测到卡住的循环)
- 所有任务标记完成

### Ralph 持续运行测试而不实施任何内容

您的 fix_plan.md 可能太模糊。使任务具体且可操作：
- 不好: "改进代码"
- 好: "为缺失的 ~/.todos.json 文件添加错误处理"

### 以后如何添加更多功能？

只需将新任务添加到 `.ralph/fix_plan.md` 并再次运行 `ralph --monitor`。Ralph 将从中断处继续。
