### 测试类别

| 类别 | 目的 | 执行速度 | 依赖 |
|----------|---------|-----------------|--------------|
| **单元** | 隔离测试单个函数 | 快 (<1s 每文件) | 无 (使用 mocks) |

## 编写测试

### pytest 基础

pytest 是测试框架。每个测试文件包含测试用例。

#### 基本测试结构

```python
"""我的功能的单元测试。"""

import pytest
from ralph_gui.models.my_model import MyModel


class TestMyModel:
    """MyModel 的测试套件。"""

    def setup_method(self):
        """每个测试前运行 setup。"""
        self.model = MyModel()

    def teardown_method(self):
        """每个测试后运行 teardown。"""
        pass

    @pytest.fixture
    def sample_data(self):
        """样本数据 fixture。"""
        return {"key": "value"}

    def test_my_feature_success(self, sample_data):
        """测试正在测试的内容的描述。"""
        result = self.model.do_something(sample_data)
        assert result is True

    def test_my_feature_failure(self):
        """测试失败情况。"""
        with pytest.raises(ValueError):
            self.model.do_something_invalid()
```

#### 使用 pytest.mark

```python
import pytest

@pytest.mark.unit
def test_fast_unit_test():
    """单元测试标记。"""
    assert True

@pytest.mark.integration
def test_integration_test():
    """集成测试标记。"""
    assert True

@pytest.mark.slow
def test_slow_operation():
    """可能在 CI 中被跳过的慢测试。"""
    pass
```

### 示例：单元测试

```python
"""CircuitBreakerModel 的单元测试。"""

import pytest
from ralph_gui.models.circuit_breaker import (
    CircuitBreakerModel,
    CircuitBreakerState
)


class TestCircuitBreakerModel:
    """CircuitBreakerModel 的测试。"""

    def setup_method(self):
        """为每个测试创建新的断路器。"""
        self.cb = CircuitBreakerModel()

    def test_initial_state_is_closed(self):
        """测试初始断路器状态为 CLOSED。"""
        assert self.cb.state == CircuitBreakerState.CLOSED

    def test_record_progress_transitions_to_closed_from_half_open(self):
        """测试进度从 HALF_OPEN 关闭电路。"""
        self.cb.state = CircuitBreakerState.HALF_OPEN
        self.cb.record_progress()
        assert self.cb.state == CircuitBreakerState.CLOSED

    def test_no_progress_in_half_open_reopens_circuit(self):
        """测试缺少进度会重新打开电路。"""
        self.cb.state = CircuitBreakerState.HALF_OPEN
        self.cb.record_no_progress()
        assert self.cb.state == CircuitBreakerState.OPEN

    @pytest.mark.parametrize("threshold,count,expected", [
        (3, 2, False),  # 尚未达到阈值
        (3, 3, True),   # 达到阈值
        (3, 4, True),   # 超过阈值
    ])
    def test_no_progress_threshold(self, threshold, count, expected):
        """测试无进度阈值检测。"""
        self.cb.config.no_progress_threshold = threshold
        for _ in range(count):
            self.cb.record_no_progress()
        assert self.cb.should_trip() == expected
```

### 示例：集成测试

```python
"""StateService 的集成测试。"""

import pytest
import tempfile
import shutil
from pathlib import Path

from ralph_gui.services.state_service import StateService
from ralph_gui.models.loop_state import LoopState


class TestStateService:
    """StateService 的集成测试。"""

    def setup_method(self):
        """为每个测试创建临时目录。"""
        self.temp_dir = tempfile.mkdtemp()
        self.state_service = StateService(self.temp_dir)

    def teardown_method(self):
        """清理临时目录。"""
        shutil.rmtree(self.temp_dir, ignore_errors=True)

    def test_save_and_load_loop_state(self):
        """测试保存和加载循环状态。"""
        loop_state = LoopState(
            loop_number=1,
            calls_made=10,
            max_calls=100,
            status="running"
        )

        self.state_service.save_loop_state(loop_state)
        loaded = self.state_service.load_loop_state()

        assert loaded.loop_number == 1
        assert loaded.calls_made == 10
        assert loaded.status == "running"

    def test_load_nonexistent_state_returns_default(self):
        """测试加载不存在的状态返回默认值。"""
        result = self.state_service.load_loop_state()
        assert result is not None
        assert isinstance(result, LoopState)
```

## 测试 Fixtures

### conftest.py

位于 `tests/conftest.py`，提供共享 fixtures：

```python
"""共享 pytest fixtures。"""

import pytest
import tempfile
import shutil
from pathlib import Path


@pytest.fixture
def temp_dir():
    """为测试创建临时目录。"""
    tmp = tempfile.mkdtemp()
    yield Path(tmp)
    shutil.rmtree(tmp, ignore_errors=True)


@pytest.fixture
def sample_ralph_project(temp_dir):
    """创建示例 Ralph 项目结构。"""
    ralph_dir = temp_dir / ".ralph"
    ralph_dir.mkdir()

    (ralph_dir / "PROMPT.md").write_text("# Test Prompt")
    (ralph_dir / "fix_plan.md").write_text("- [ ] Test task")
    (ralph_dir / "AGENT.md").write_text("# Test Agent")

    return temp_dir


@pytest.fixture
def mock_claude_response():
    """Mock Claude CLI 响应。"""
    return {
        "status": "success",
        "sessionId": "test-session-123",
        "result": {
            "content": [{"type": "text", "text": "Test response"}]
        }
    }


@pytest.fixture
def circuit_breaker_config():
    """默认断路器配置。"""
    from ralph_gui.models.circuit_breaker import CircuitBreakerConfig
    return CircuitBreakerConfig(
        no_progress_threshold=3,
        same_error_threshold=5,
        output_decline_threshold=70
    )
```

## 运行测试

### 运行所有测试

```bash
# 带详细输出的所有测试
pytest -v

# 带覆盖率
pytest --cov=src/ralph_gui --cov-report=html
```

### 按类别运行

```bash
# 仅单元测试
pytest tests/unit/

# 仅集成测试
pytest tests/integration/

# 带特定标记
pytest -m unit
pytest -m integration
```

### 运行特定测试

```bash
# 单个测试文件
pytest tests/unit/test_circuit_breaker.py

# 按名称单个测试
pytest tests/unit/test_circuit_breaker.py::TestCircuitBreakerModel::test_initial_state_is_closed

# 使用 -k 进行模式匹配
pytest -k "circuit_breaker"
pytest -k "test_initial"
```

### 调试模式

```bash
# 首次失败时停止
pytest -x

# 失败时显示局部变量
pytest -l

# 启用 PDB 失败
pytest --pdb

# 禁用输出捕获
pytest -s
```

## CI/CD 集成

### GitHub Actions 工作流

```yaml
name: Test Suite

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

jobs:
  test:
    runs-on: windows-latest

    steps:
    - uses: actions/checkout@v4

    - name: Setup Python
      uses: actions/setup-python@v5
      with:
        python-version: '3.10'

    - name: Install dependencies
      run: |
        pip install -r requirements.txt
        pip install pytest pytest-cov pytest-mock

    - name: Run unit tests
      run: pytest tests/unit/ -v

    - name: Run integration tests
      run: pytest tests/integration/ -v

    - name: Run with coverage
      run: pytest --cov=src/ralph_gui --cov-report=xml
```

## 故障排除

### 测试失败

#### 阅读 pytest 输出

```
FAILED tests/unit/test_circuit_breaker.py::TestCircuitBreakerModel::test_initial_state_is_closed
AssertionError: assert 'CLOSED' == 'OPEN'
```

- **FAILED**: 测试失败
- **test_initial_state_is_closed**: 测试名称
- **AssertionError**: 失败类型
- **assert 'CLOSED' == 'OPEN'**: 预期 vs 实际

#### 调试步骤

1. **运行单个测试**:
   ```bash
   pytest tests/unit/test_circuit_breaker.py::test_initial_state_is_closed -v
   ```

2. **添加调试输出**:
   ```python
   def test_debug(self):
       result = my_function()
       print(f"Result: {result}")  # 用 -s 在输出中显示
       assert result == expected
   ```

3. **使用断点**:
   ```python
   def test_debug(self):
       import pdb; pdb.set_trace()
       result = my_function()
   ```

### Mock 问题

#### Mock 未被调用

```python
# 验证 mock 被调用
from unittest.mock import Mock, call

mock_service = Mock()
mock_service.do_something()

mock_service.do_something.assert_called_once()
mock_service.do_something.assert_called_with(expected_arg)
```

#### 错误的 Mock 响应

```python
# 设置特定的 mock 返回值
mock_service = Mock()
mock_service.get_data.return_value = {"key": "value"}

# 或使用 side_effect 获取不同响应
mock_service.get_data.side_effect = [
    {"first": "call"},
    {"second": "call"}
]
```
